# **🧩 Atividade PBL – Aula 15**

**Modelos de Maturidade – LocalEats**

## **📌 Contexto**

Na aula anterior, a equipe analisou o processo utilizado para desenvolver o LocalEats.

Agora o objetivo é avaliar o nível de maturidade desse processo, identificando pontos fortes, lacunas e oportunidades de melhoria.

Modelos como CMMI e MPS.BR ajudam organizações a compreender o quanto seus processos são definidos, gerenciados e continuamente aprimorados.

👉 Link para o sistema LocalEats: [https://local-eats-unisenac.vercel.app/](https://local-eats-unisenac.vercel.app/)

## **🎯 Objetivo da Atividade**

Aplicar os conceitos de:

* Maturidade de processos  
* CMMI e MPS.BR  
* Diagnóstico organizacional  
* Identificação de oportunidades de melhoria  
* Evolução da qualidade de processo

👉 O foco não é avaliar o código do sistema.

👉 O objetivo é analisar a maturidade do processo de desenvolvimento utilizado pela equipe.

## **📝 Tarefas**

Elaborem um documento contendo:

### **🔹 1\. Diagnóstico de Maturidade**

Avaliem o processo da equipe respondendo à tabela abaixo:

| Critério | Sim | Parcial | Não |
| ----- | ----- | ----- | ----- |
| Os requisitos são documentados? |  |  |  |
| Existe controle de mudanças? |  |  |  |
| Há atividades de teste definidas? |  |  |  |
| Os defeitos são registrados? |  |  |  |
| O processo de desenvolvimento é conhecido por toda a equipe? |  |  |  |
| As tarefas são planejadas e acompanhadas regularmente? |  |  |  |
| Existe padronização para implementação de funcionalidades? |  |  |  |
| Os testes são executados antes da entrega das funcionalidades? |  |  |  |
| Há revisão de código ou validação por outro integrante da equipe? |  |  |  |
| A equipe utiliza ferramentas para gerenciamento das atividades? |  |  |  |
| Os artefatos do projeto (requisitos, testes, código) são organizados e versionados? |  |  |  |
| Existe rastreabilidade entre requisitos e funcionalidades implementadas? |  |  |  |
| A equipe realiza reuniões ou momentos de retrospectiva para identificar melhorias? |  |  |  |
| Existem indicadores ou métricas para acompanhar a qualidade do projeto? |  |  |  |

Ao final, classifiquem o processo como:

* Inicial  
* Gerenciado  
* Definido  
* Quantitativamente Gerenciado  
* Em Otimização

Justifiquem a classificação em até 10 linhas.

### **🔹 2\. Identificação de Lacunas**

Listem pelo menos 3 pontos que podem ser melhorados no processo atual.

Exemplo:

| Lacuna | Impacto |
| ----- | ----- |
| Falta de métricas | Dificulta acompanhamento da qualidade |

### **🔹 3\. Propostas de Melhoria**

Proponham pelo menos 3 ações para aumentar a maturidade do processo.

Exemplo:

| Melhoria | Benefício |
| ----- | ----- |
| Padronizar testes | Maior confiabilidade |
