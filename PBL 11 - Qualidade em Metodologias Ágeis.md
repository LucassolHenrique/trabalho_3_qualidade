# **🧩 Atividade PBL – Aula 16**

**Qualidade em Metodologias Ágeis – LocalEats**

## **📌 Contexto**

Nas aulas anteriores, a equipe mapeou o processo de desenvolvimento utilizado no projeto LocalEats e avaliou seu nível de maturidade.

Agora o objetivo é analisar esse processo sob a perspectiva das metodologias ágeis, identificando práticas já utilizadas e propondo melhorias que possam contribuir para entregas mais organizadas, colaborativas e com maior qualidade.

👉 Link para o sistema LocalEats: [https://local-eats-unisenac.vercel.app/](https://local-eats-unisenac.vercel.app/)

## **🎯 Objetivo da Atividade**

Aplicar os conceitos de:

* Scrum  
* XP (Extreme Programming)  
* Kanban  
* Lean Software Development  
* Definition of Ready (DoR)  
* Definition of Done (DoD)  
* Qualidade em metodologias ágeis

👉 O foco não é avaliar o código-fonte do sistema.

👉 O objetivo é analisar e aprimorar o processo de desenvolvimento do projeto.

## **📝 Tarefas**

Considerando o processo de desenvolvimento utilizado pela equipe, analisem a presença das seguintes práticas ágeis:

| Controle visual das atividades |  |  |  |
| Melhoria contínua |  |  |  |

Ao final, escrevam uma conclusão (até 10 linhas) destacando os principais pontos fortes e oportunidades de melhoria do processo.

Proponham pelo menos **4 melhorias** para o processo atual utilizando conceitos estudados em aula.

As melhorias podem ser inspiradas em práticas de Scrum, XP, Kanban, Lean ou outras abordagens ágeis discutidas durante a aula.

### **🔹 3\. Definition of Ready (DoR)**

Elaborem uma Definition of Ready para o projeto.

Definam pelo menos **5 critérios** que uma funcionalidade deve atender antes de entrar em desenvolvimento.

**Exemplo de critério:**

* O requisito possui critérios de aceitação definidos.

### **🔹 4\. Definition of Done (DoD)**

Elaborem uma Definition of Done para o projeto.

Definam pelo menos **5 critérios** para considerar uma funcionalidade concluída.

**Exemplo de critério:**

* Os critérios de aceitação da funcionalidade foram atendidos.

## **📊 Avaliação (Rubrica – Unisenac-RS)**

### **🔴 D — Não atingiu**

* Atividade incompleta.  
* Pouca relação com os conceitos estudados.  
* DoR ou DoD ausentes.

### **🟡 C — Parcial**

* Análise superficial do processo.  
* Melhorias pouco justificadas.  
* Critérios genéricos de DoR e DoD.

### **🔵 B — Pleno**

* Boa análise das práticas ágeis.  
* Melhorias coerentes com o contexto do projeto.  
* DoR e DoD adequados e aplicáveis.

### **🟢 A — Excelência**

* Análise crítica e consistente do processo.  
* Melhorias relevantes e bem justificadas.
